## 诊断报告 (样本ID: sample_02_logic_error)

### 1. 核心问题
- **错误类型**：边界条件/逻辑错误
- **具体描述**：代码没有处理空列表的情况，当输入为空列表时会导致除以零错误（ZeroDivisionError）。

### 2. 错误原因分析
- 学生忽略了边界条件的处理，假设输入总是非空的。
- 这是典型的"防御性编程"缺失，没有考虑所有可能的输入情况。
- 在数学上，空列表的平均值没有定义，但在编程中需要明确处理。

### 3. 修复步骤与建议
1. **添加边界检查**：在计算之前检查列表是否为空。
2. **定义处理策略**：
   - 返回一个特殊值（如0或None）
   - 抛出有意义的异常（如ValueError）
   - 返回NaN（如果使用数学库）
3. **代码修改示例**：
   ```python
   def calculate_average(numbers):
       if not numbers:  # 或 len(numbers) == 0
           return 0  # 或者 raise ValueError("列表不能为空")
       return sum(numbers) / len(numbers)
### 4. 教学提示与延伸思考
可提问引导："如果用户传递一个空列表，你的代码会发生什么？如何避免这种情况？"

知识延伸：讨论Python中的异常处理（try-except）和防御性编程。

相关练习：其他需要考虑边界条件的函数（如最大值、最小值、中位数等）。

测试思维：强调编写测试用例时要包含边界情况（空列表、单个元素、负数等）。

### 5. 错误预防建议
总是考虑函数的边界条件。

使用类型提示和文档字符串说明函数的预期输入。

编写单元测试覆盖边界情况。

使用断言（assert）在开发阶段捕获问题。
